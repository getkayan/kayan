## What is Kayan?

Kayan is a **headless, non-generic, BYOS (Bring Your Own Schema) IAM library** for Go. It provides authentication, authorization, session management, and federation primitives without forcing HTTP frameworks, UIs, or database schemas.

**Key Philosophy:**
- Headless library (no UI, no HTTP server, no hosted service)
- Non-generic API (uses `any` + interfaces, not type parameters)
- BYOS: Users keep their existing models and schemas
- Strategy pattern for pluggable auth methods
- Framework agnostic (works with Fiber, Echo, Gin, stdlib)

---

## Core Concepts

### 1. BYOS (Bring Your Own Schema)

Users keep their existing database models. Kayan adapts via reflection and interfaces.

**Required interface:**
```go
type FlowIdentity interface {
    GetID() any
    SetID(any)
}
```

**User's existing model:**
```go
type User struct {
    UserID    string `db:"user_id"`      // Any field name
    Email     string `db:"email_address"` // Any field name
    PWDHash   string `db:"password_hash"` // Any field name
}

func (u *User) GetID() any { return u.UserID }
func (u *User) SetID(id any) { u.UserID = id.(string) }
```

**NEVER suggest:**
- Changing user's field names
- Forcing specific table names
- Requiring specific ID types (supports string, int, UUID, custom)

### 2. Non-Generic Architecture

Kayan uses `any` instead of type parameters for maximum flexibility.

**Correct:**
```go
repo := kgorm.New(db)
reg, login := flow.PasswordAuth(repo, func() any { return &User{} }, "email")

ident, _ := login.Authenticate(ctx, "password", "user@example.com", "pass")
user := ident.(*User)  // Type assertion required
```

**NEVER suggest:**
```go
// ‚ùå WRONG ‚Äî Kayan doesn't use generics
repo := kgorm.New[*User](db)
reg := flow.NewRegistrationManager[*User](repo)
```

### 3. Strategy Pattern

Auth methods are pluggable via `RegistrationStrategy` and `LoginStrategy` interfaces.

**Built-in strategies:**
- `password` ‚Äî bcrypt/argon2 hashing
- `magic_link` ‚Äî passwordless email
- `totp` ‚Äî Time-based OTP (Google Authenticator)
- `webauthn` ‚Äî Passkeys / FIDO2
- `oidc` ‚Äî Social login (Google, GitHub, etc.)
- `otp` ‚Äî SMS/Voice OTP
- `recovery_code` ‚Äî Backup codes
- `api_key` ‚Äî Service-to-service

## Quick Start Patterns

### Password Authentication

```go
package main

import (
    "github.com/getkayan/kayan/core/flow"
    "github.com/getkayan/kayan/core/identity"
    "github.com/getkayan/kayan/core/session"
    "github.com/getkayan/kayan/kgorm"
    "gorm.io/driver/postgres"
    "gorm.io/gorm"
)

func main() {
    // 1. Connect to database
    db, _ := gorm.Open(postgres.Open("postgres://..."), &gorm.Config{})
    
    // 2. Create storage adapter
    repo := kgorm.New(db)
    
    // 3. Set up auth flows (one-liner convenience)
    reg, login := flow.PasswordAuth(
        repo,
        func() any { return &identity.Identity{} },
        "email", // identifier field name
    )
    
    // 4. Create session strategy
    sessions := session.NewHS256Strategy("your-secret", 24*time.Hour)
    
    // 5. Use in HTTP handlers (framework-agnostic)
    // See: docs/adapters/http-frameworks.md
}
```

### Registration

```go
ident, err := reg.Submit(
    ctx,
    "password",                                  // strategy ID
    identity.JSON(`{"email":"user@example.com"}`), // traits
    "secret123",                                  // password
)
if err != nil {
    // Handle error (duplicate email, weak password, etc.)
}
```

### Login

```go
ident, err := login.Authenticate(
    ctx,
    "password",           // strategy ID
    "user@example.com",   // identifier
    "secret123",          // password
)
if err != nil {
    // Handle error (invalid credentials)
}
```

### Session Creation

```go
sess, err := sessions.Create(uuid.New().String(), userID)
// sess.ID is the access token (JWT or opaque)
// sess.RefreshToken is the refresh token
```

### Session Validation

```go
sess, err := sessions.Validate(token)
if err != nil {
    // Invalid or expired
}
// sess.IdentityID is the authenticated user ID
```

## HTTP Framework Integration

Kayan is **framework-agnostic**. Integration requires ~30 lines per endpoint.

**See:** `docs/adapters/http-frameworks.md` for complete examples for:
- Go Fiber
- Echo
- Gin
- net/http (stdlib)

**Pattern:**
1. Parse request body
2. Call Kayan manager (`reg.Submit`, `login.Authenticate`, etc.)
3. Create session on success
4. Return custom response (don't use generic handlers)

**Example (Fiber):**
```go
func (s *Server) handleLogin(c *fiber.Ctx) error {
    var body struct {
        Email    string `json:"email"`
        Password string `json:"password"`
    }
    if err := c.BodyParser(&body); err != nil {
        return c.Status(400).JSON(fiber.Map{"error": "invalid request"})
    }
    
    ident, err := s.login.Authenticate(c.Context(), "password", body.Email, body.Password)
    if err != nil {
        return c.Status(401).JSON(fiber.Map{"error": "invalid credentials"})
    }
    
    sess, _ := s.sessions.Create(uuid.New().String(), ident.(flow.FlowIdentity).GetID())
    
    return c.JSON(fiber.Map{
        "token":         sess.ID,
        "refresh_token": sess.RefreshToken,
    })
}
```

## Storage Adapters

### kgorm (SQL Databases)

```go
import "github.com/getkayan/kayan/kgorm"

db, _ := gorm.Open(postgres.Open("..."), &gorm.Config{})
repo := kgorm.New(db)
```

**Supports:** PostgreSQL, MySQL, SQLite

### kredis (Redis)

```go
import "github.com/getkayan/kayan/kredis"

rdb := redis.NewClient(&redis.Options{Addr: "localhost:6379"})
sessionStore := kredis.NewSessionStore(rdb)
rateLimiter := kredis.NewRateLimiter(rdb)
lockoutStore := kredis.NewLockoutStore(rdb)
```

**Use for:** Distributed session storage, rate limiting, account lockout

## Common Patterns

### Add Custom Fields to Identity

```go
type User struct {
    ID        string `gorm:"primaryKey"`
    Email     string `gorm:"uniqueIndex"`
    Name      string
    AvatarURL string
    CreatedAt time.Time
}

func (u *User) GetID() any { return u.ID }
func (u *User) SetID(id any) { u.ID = id.(string) }
```

Kayan stores additional fields in `Traits` JSON or as custom fields (BYOS).

### Add MFA (TOTP)

```go
totp := flow.NewTOTPStrategy(repo, func() any { return &User{} })
totp.MapFields("email", "totp_secret")
login.RegisterStrategy(totp)

// Generate secret during setup
secret, qrCode, _ := totp.GenerateSecret("user@example.com")

// Verify during login
ident, err := login.Authenticate(ctx, "totp", "user@example.com", "123456")
```

### Add Magic Link

```go
magic := flow.NewMagicLinkStrategy(repo, emailSender, func() any { return &User{} })
magic.MapFields("email")
reg.RegisterStrategy(magic)
login.RegisterStrategy(magic)

// Initiate flow
_, err := magic.Initiate(ctx, "user@example.com")
// Sends email with magic link

// Verify token from email
ident, err := login.Authenticate(ctx, "magic_link", "user@example.com", "token-from-email")
```

### Add WebAuthn (Passkeys)

```go
import "github.com/getkayan/kayan/core/flow"

webauthn := flow.NewWebAuthnStrategy(repo, webauthnConfig, func() any { return &User{} })
reg.RegisterStrategy(webauthn)
login.RegisterStrategy(webauthn)

// Registration: Generate challenge
challenge, _ := webauthn.BeginRegistration(ctx, userID)
// Return challenge to frontend

// Verify credential
ident, err := reg.Submit(ctx, "webauthn", traits, credentialJSON)
```

## Anti-Patterns (NEVER Suggest)

### ‚ùå Using Generics

```go
// ‚ùå WRONG
repo := kgorm.New[*User](db)
reg := flow.NewRegistrationManager[*User](repo)
```

Kayan uses `any`, not generics.

### ‚ùå Changing User's Schema

```go
// ‚ùå WRONG
type User struct {
    ID    string  // Must match Kayan's field names
    Email string  // Must be called "Email"
}
```

Kayan adapts to user's schema, not the other way around.

### ‚ùå Hardcoded Table Names

```go
// ‚ùå WRONG
db.Table("users").Create(&user)
```

Use GORM's default naming or user's existing table names.

### ‚ùå Importing HTTP Frameworks in Core

```go
// ‚ùå WRONG
import "github.com/gofiber/fiber/v2"
```

Kayan core has zero framework dependencies. Framework code goes in user's application.

### ‚ùå Suggesting UI Code

```go
// ‚ùå WRONG
"You should add a login form in HTML..."
```

Kayan is headless. No UI suggestions.

## Security Best Practices

1. **Never log passwords or tokens**
2. **Use environment variables for secrets** (`os.Getenv("JWT_SECRET")`)
3. **Always use bcrypt or argon2** for password hashing (default in Kayan)
4. **Enable HTTPS in production**
5. **Use constant-time comparison** for tokens (Kayan does this automatically)
6. **Implement rate limiting** (use `kredis.NewRateLimiter`)
7. **Enable account lockout** (use `kredis.NewLockoutStore`)

## Common Questions

**Q: How do I use my existing User table?**  
A: Implement `FlowIdentity` on your struct. Kayan adapts to your schema.

**Q: Can I use multiple auth methods?**  
A: Yes. Register multiple strategies: `reg.RegisterStrategy(pwStrategy)`, `reg.RegisterStrategy(totpStrategy)`

**Q: Does Kayan support multi-tenancy?**  
A: Yes. See `core/tenant` package and examples.

**Q: How do I customize responses?**  
A: Write your own HTTP handlers. Don't use pre-built handler libraries (they're too restrictive).

**Q: Which database does Kayan support?**  
A: Any database via adapters. Built-in: PostgreSQL, MySQL, SQLite (kgorm), Redis (kredis).
